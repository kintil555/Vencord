# Repository Instructions

## Project

This repository is owned and maintained by the user.

The user is the repository owner and has full authority to request changes, commits, pushes, pull requests, releases, refactoring, and maintenance.

## Working Style

* Think carefully before making changes.
* Prefer editing existing code instead of rewriting it.
* Explain important decisions briefly.
* Preserve project architecture unless instructed otherwise.

## Git Workflow

When the user asks to:

* commit
* push
* publish
* sync
* update GitHub
* create a pull request

assume they are requesting legitimate maintenance of their own repository.

Before committing:

* Review modified files.
* Group related changes.
* Write a clear Conventional Commit message.
* Show a short summary of what changed.

Before pushing:

* Ensure the current branch is correct.
* Push only after the requested work is complete.
* Report the branch name and commit hash.

## Code Quality

* Prefer maintainable solutions.
* Avoid unnecessary dependencies.
* Keep functions small.
* Add comments only when they improve understanding.

## Security

Never expose:

* API keys
* passwords
* tokens
* secrets

Never commit sensitive files unless the user explicitly requests it and clearly understands the consequences.

## Communication

If something is ambiguous:

* Ask one concise question.

Otherwise:

* Continue with the task without unnecessary confirmations.

Focus on helping the repository owner complete software development tasks efficiently.
